## 2. 核心数据库 Schema (PostgreSQL/pgvector)

根据 PRD 3.4 节的“记忆网格”和 3.2.2 节的“规划编排器”，推荐以下表结构。

```sql
-- 1. 记忆网格表 (Memory Mesh)
-- 整合冷、温记忆，支持向量检索
CREATE EXTENSION IF NOT EXISTS vector;

CREATE TABLE memories (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id VARCHAR(64) NOT NULL,
    user_id VARCHAR(64) NOT NULL,
    role_name VARCHAR(32), -- 产生该记忆的角色（如 architect, engineer）
    content TEXT NOT NULL,
    embedding vector(1536), -- OpenAI/Claude 2026 标准向量维度
    metadata JSONB DEFAULT '{}', -- 存储 PRD 中提到的 Context 对象
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_memories_session_id (session_id),
    INDEX idx_memories_vector USING hnsw (embedding vector_cosine_ops)
);

-- 2. 任务状态机表 (Task State Machine)
-- 对应 PRD 3.2.2 节的规划格式
CREATE TYPE task_status AS ENUM ('pending', 'in_progress', 'completed', 'failed', 'retrying');

CREATE TABLE tasks (
    task_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    plan_id UUID NOT NULL,
    parent_task_id UUID, -- 支持子任务拆解
    description TEXT NOT NULL,
    assignee VARCHAR(32) NOT NULL, -- 指派的角色
    status task_status DEFAULT 'pending',
    input_params JSONB,
    output_data JSONB,
    retry_count INT DEFAULT 0,
    max_retries INT DEFAULT 3,
    dependencies UUID[] DEFAULT '{}', -- 依赖的 task_id 列表
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- 3. 技能注册表 (Skill Registry)
-- 对应 PRD 3.3.1 节
CREATE TABLE skill_registry (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(64) UNIQUE NOT NULL,
    version VARCHAR(16),
    manifest JSONB NOT NULL, -- 存储完整 Skill Manifest (JSON Schema)
    is_active BOOLEAN DEFAULT TRUE,
    last_called_at TIMESTAMP WITH TIME ZONE
);
```

## 3. 核心 API 接口初步定义 (FastAPI)

遵循 PRD 4.2 节的层间通信协议。

### A. 意图与路由 (L1 -> L2)

**POST** `/api/v1/intent/process`

- **Request Body**:
  ```json
  {
    "raw_input": "帮我设计一套用户认证系统的 API",
    "platform": "slack",
    "user_id": "U12345",
    "context": { "session_id": "S987", "project_id": "P55" }
  }
  ```
- **Response**: 触发角色工厂实例化，返回 `TaskRequest`。

### B. 任务编排与执行 (L2 -> L3)

**POST** `/api/v1/orchestrator/dispatch`

- **Logic**: 接收意图，生成任务计划并存入 `tasks` 表。
- **Interface**: 异步接口，返回 `plan_id`，用户通过 WebSocket 或轮询获取状态。

### C. 工具执行 (MCP 适配器)

**POST** `/api/v1/tools/execute`

- **Request Body** (遵循 MCP 规范):
  ```json
  {
    "method": "tools/call",
    "params": {
      "name": "read_file",
      "arguments": { "path": "docs/prd.md" }
    },
    "agent_token": "..."
  }
  ```

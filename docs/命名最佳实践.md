### 模块命名理由与最佳实践说明：

1. **`src/access/`** **(L1)**：不叫 `input`，因为它是双向的。**适配器模式**（Adapters）确保了无论用户从哪个平台接入，核心逻辑看到的数据都是一致的。
2. **`src/core/`** **(L2)**：这是大脑所在地。\*\*状态机（State）\*\*独立出来是因为在 Agent 开发中，任务状态的持久化是恢复中断会话的关键。
3. **`src/execution/`** **(L3)**：强调“执行”。特别设立了 **`mcp/`** 文件夹，这标志着你的项目支持 2026 年主流的 **Model Context Protocol**，可以轻松接入现有的外部 AI 工具。
4. **`src/persistence/`** **(L4)**：不只叫 `database`。因为它包含了 **Memory Mesh**（记忆网格）的复杂逻辑，涉及向量检索和短期上下文切换。
5. **`src/workflow/`** **(L5)**：这是你 PRD 的精华。**`loop.py`** 专门负责“执行 -> 报错 -> 修复”的闭环，这是区分普通 Agent 和高级 Agent 的核心模块。
6. **独立** **`skills/`** **目录**：遵循 **“技能原子化”** 最佳实践。这里的每个子文件夹未来都可以独立打包成一个 MCP Server。
